# Strategy Writer Skill

B2B SaaS / 플랫폼 비즈니스의 전략 문서를 **대화형으로 수집 → 구조화 → 작성 → 노션 저장**하는 워크플로우.

GTM 전략 문서의 표준 7개 섹션을 기반으로 하며, 각 섹션을 순서대로 채워나가는 방식으로 작성한다.

```
1. Market Context     시장 현황과 진입 근거
2. ICP & Segmentation 타겟 고객 정의
3. Positioning        가치 제안과 메시지
4. GTM Motion         영업/마케팅 방식 선택
5. Channel & Sales    채널과 영업 전략
6. Success Metrics    성공 기준과 측정 지표
7. Open Questions     미결 과제와 검증 항목
```

---

## 핵심 원칙

### 1. 질문 우선, 작성 나중
섹션 단위로 정보를 수집한 뒤 작성한다. 정보가 부족한 채로 섹션을 작성하지 않는다.

### 2. ask_user_input 활용
선택지로 답할 수 있는 질문은 반드시 `ask_user_input_v0` 툴로 제시한다. 채팅으로 질문을 나열하지 않는다. 단, 자유입력이 필요한 경우(선택지로 담기 어려운 경우)는 채팅으로 질문한다.

### 3. 리서치 병행
질문 전에 web_search로 관련 기업/시장을 사전 리서치한다. 정보 없이 묻지 않는다 — 리서치 결과와 함께 의견을 제시하고, 판단을 돕는다.

### 4. 가설 명시
추론이나 리서치에 기반한 판단은 반드시 문서 내 `⚠️ 가설 — 검증 필요`로 표기한다.

### 5. 구체성이 전부
막연한 문장은 숫자, 이름, 조건으로 바꾼다.

| 나쁜 예 | 좋은 예 |
|---|---|
| "광고 수익을 높여줍니다" | "IAA 매출 평균 80% 이상 증분 (논게임 고객사 기준)" |
| "타겟 고객은 중소 게임사" | "T1~T2 국내 게임사, 서브 타이틀 DAU 5만~20만" |
| "아웃바운드 영업을 진행합니다" | "사업팀장 또는 대표 직접 접근, 서브 타이틀 테스트 제안 구조" |

## Stage 1: 문서 유형 및 배경 파악

작성 시작 전 아래를 파악한다. 대화에서 이미 주어진 맥락이 있으면 추출하고, 빠진 것만 묻는다.

1. **어떤 전략인가?** GTM / 포지셔닝 / 경쟁 / 시장 확장 / 파트너십
2. **회사와 제품 컨텍스트는?** 현재 포지션, 핵심 강점, 기존 고객 레퍼런스
3. **이 전략의 목적은?** 신규 시장 진입 / 기존 시장 방어 / 새로운 세그먼트 공략
4. **참고할 자료가 있는가?** 노션 링크, 기존 전략 문서, 경쟁사 분석

> 노션 링크가 있으면 `notion-fetch`로 직접 읽어온다.

## Stage 2: 섹션별 정보 수집 및 작성 가이드

### 섹션 1. Market Context

**목표**: 이 전략이 왜 지금 필요한가를 한눈에 보여준다.
독자가 이 섹션만 읽어도 "아, 이래서 이 시장에 지금 들어가는 거구나"를 이해해야 한다.

수집할 정보:
- 현재 회사 포지션과 성장 한계
- 타겟 시장의 규모와 기회
- 경쟁 구도 (주요 플레이어, 공백 영역)
- 지금 진입하는 이유 (타이밍 근거)

작성 구조:
```
### 현재 포지션
[회사가 지금 어디 있는지, 왜 한계에 도달했는지]

### 시장 기회
[타겟 시장 규모, 경쟁 공백, 진입 근거]

### 경쟁 구도
[주요 플레이어와 다로의 차별점]
```

**체크**:
- 진입 근거가 데이터 또는 관찰 가능한 사실에 기반하는가?
- "시장이 크다"가 아니라 "왜 우리가 지금 이 시장에서 이길 수 있는가"가 보이는가?

---

### 섹션 2. ICP & Segmentation

**목표**: "가장 먼저 공략할 고객"을 한 줄로 말할 수 있어야 한다.

ICP는 완성되는 문서가 아니다. 고객 데이터가 쌓일수록 업데이트해야 한다. (First Round, Emery Rosansky)

ICP 도출 3단계 (정보가 부족할 때 이 순서로 수집한다):
1. **기존 고객 패턴 분석** — 가장 빠르게 계약한 곳, 가장 큰 딜, 이탈 없는 고객의 공통점은?
2. **제외 대상 명확화** — 어떤 고객이 성과를 내기 어려운가? 이유는?
3. **Operational Fit 가설** — 오늘 당장 우리 솔루션이 우선순위가 될 조건은? (특정 상황, 규모, 기술 스택 등)

수집할 정보:
- 타겟 세그먼트 (지역, 산업, 규모, 장르/카테고리)
- 핵심 페인포인트
- 제외 조건 (이유 포함)
- 의사결정자 구조
- 앵커 케이스 (첫 고객 후보)와 보류 케이스

작성 구조:
```
### 핵심 ICP 프로파일
[한 줄 정의 + 표: 항목 / 정의]

### 제외 대상
[표: 장르/카테고리 / 이유]

### 앵커 케이스
[구체적인 첫 고객 후보, 왜 이 고객인가]
```

**체크**:
- ICP를 "한 줄"로 말할 수 있는가?
- 제외 조건에 이유가 명시되어 있는가?
- 앵커 케이스가 구체적인가? (회사명 / 타이틀 / DAU / 진입 경위)

---

### 섹션 3. Positioning

**목표**: 타겟 고객이 느끼는 페인포인트에서 출발해, 우리가 제공하는 가치를 한 줄로 표현한다.

포지셔닝의 3가지 구성 요소:
- **페인포인트**: 고객이 지금 무엇 때문에 고통받는가
- **가치 제안**: 우리가 그 고통을 어떻게 해결하는가 (수치 포함)
- **차별점**: 경쟁 대비 왜 우리인가

수집할 정보:
- 핵심 가치 제안 메시지 (1순위 메시지가 무엇인가)
- 사용 금지 표현 / 포지셔닝 (고객에게 오해를 줄 수 있는 것)
- 기존 레퍼런스를 어떻게 활용할 것인가
- 주요 의구심과 대응 방법

작성 구조:
```
### 핵심 포지셔닝 (한 줄)
[고객 페인포인트 → 우리의 해결 → 수치 근거]

### 메시지 구조
[표: 레이어 / 메시지 / 강도]

### 주요 의구심 대응
[고객이 가장 먼저 던질 반론과 우리의 답변]

### 포지셔닝 주의사항
[사용 금지 표현, 오해를 줄 수 있는 프레이밍]
```

**체크**:
- 핵심 메시지에 수치가 포함되어 있는가?
- "우리가 좋다"가 아니라 "고객에게 무엇이 달라지는가"로 쓰여 있는가?
- 포지셔닝 주의사항이 명시되어 있는가?

---

### 섹션 4. GTM Motion

**목표**: Sales-Led / PLG / Hybrid 중 어떤 방식으로 고객을 획득할지를 명확히 선택한다.
Motion이 결정되면 이후 채널 선택, 영업 프로세스, 인력 구성이 모두 여기서 파생된다. (a16z GTM Boot Camp)

| 모션 | 특징 | 적합 조건 |
|---|---|---|
| **Sales-Led** | 영업팀 주도. 긴 사이클. 높은 ACV | 복잡한 솔루션, 운영 대행, 프리미엄 B2B 서비스 |
| **Product-Led (PLG)** | 제품이 스스로 확산. 셀프서브 온보딩 | 낮은 ACV, 대량 사용자, SaaS 툴 |
| **Hybrid** | PLG로 유입 + 영업으로 전환 | Mid-market, 프리미엄 업셀 필요 |

수집할 정보:
- ACV(계약 단가) 수준
- 의사결정 사이클 길이
- 제품 자체의 셀프서브 가능 여부
- 현재 팀의 영업 역량

작성 구조:
```
### 선택한 Motion
[Sales-Led / PLG / Hybrid + 선택 근거]

### Motion 근거
[ACV, 사이클 길이, 제품 특성 기반 판단]
```

**체크**:
- Motion 선택 근거가 명시되어 있는가?
- Motion에 맞지 않는 채널이 이후 섹션에 포함되지 않았는가?

---

### 섹션 5. Channel & Sales Strategy

**목표**: 어디서, 어떻게 고객을 만나고, 어떻게 계약을 닫는가.

수집할 정보:
- 주요 획득 채널 (인바운드 / 아웃바운드 / 파트너십 / 컨퍼런스)
- 첫 미팅 구조와 딜 제안 방식
- 의사결정자 접근 방법
- 초기 케이스스터디 확보 경로

작성 구조:
```
### 채널 전략
[표: 채널 / 내용 / 우선순위]

### 영업 진입 전략
[첫 접근 방식, 딜 구조, 의사결정자 타겟팅]

### 케이스스터디 확보 경로
[첫 성공 사례를 만드는 구체적 경로]
```

**체크**:
- 채널이 앞서 선택한 GTM Motion과 일치하는가?
- 케이스스터디 확보 경로가 구체적인가? (회사명, 타이틀, 접근 방식)
- 플랜 B가 명시되어 있는가?

---

### 섹션 6. Success Metrics

**목표**: 이 전략이 성공했는지 어떻게 판단할 것인가.

| 지표 유형 | 설명 | 예시 |
|---|---|---|
| **Leading** | 성과를 예측하는 선행 지표 | 미팅 수, 파이프라인 생성 |
| **Lagging** | 결과를 확인하는 후행 지표 | 계약 수, 매출, NRR |
| **Health** | 고객 관계 건강도 | 케이스스터디 확보 수, 갱신율 |

수집할 정보:
- 이 전략의 성공 기준 (3개월 / 6개월 / 12개월)
- 측정 가능한 수치 목표
- 실패 신호 (언제 전략을 바꿔야 하는가)

작성 구조:
```
### 핵심 지표
[표: 지표 / 유형 / 목표치 / 측정 시점]

### 실패 신호
[이 시점까지 이 수치가 안 나오면 전략을 재검토한다]
```

**체크**:
- 지표가 측정 가능한가? "성공적인 운영" 같은 추상 표현 금지
- 실패 신호가 명시되어 있는가?
- 목표치에 근거(벤치마크 또는 과거 데이터)가 있는가?

---

### 섹션 7. Open Questions

**목표**: 아직 모르는 것, 검증이 필요한 것을 숨기지 않고 드러낸다.
PRD 원칙과 동일 — 리뷰어가 빈칸을 발견하는 것보다, 질문이 명시된 문서가 낫다.

작성 구조:
```
| 항목 | 내용 | 우선순위 |
|---|---|---|
| [검증 필요 항목] | [무엇을 확인해야 하는가] | 🔴 / 🟡 / 🟢 |
```

**체크**:
- 가설로 작성된 내용이 모두 이 섹션에 트래킹되고 있는가?
- 우선순위가 명시되어 있는가?

## Stage 5: 노션 저장

노션 페이지 URL이 주어진 경우:
1. `notion-fetch`로 현재 페이지 상태 확인
2. 빈 페이지 → `replace_content`로 전체 작성
3. 기존 내용 있음 → 섹션별로 `update_content` 사용
4. 저장 완료 후 페이지 URL 전달

노션 저장 시 주의사항:
- Notion-flavored Markdown 문법 준수
- 콜아웃: `<callout icon="⚠️">내용</callout>`
- 토글: `<details><summary>제목</summary>내용</details>`

## 도메인 레퍼런스 (AdTech / 모바일 광고)

**DARO 제품 아키텍처, 전략 3축, AdTech 전체 용어집**은 `prd-writer` 스킬의 'DARO 공유 컨텍스트' 섹션을 참조합니다. 아래는 전략 문서에서 추가로 자주 쓰이는 비즈니스 지표만 정리합니다.

| 용어 | 정의 |
